众所周知，S切对自己做出的料理的装饰是很讲究的。她的新烧饼料理有一个画有笑脸的一面，也有一面没有任何装饰。
这次，许多新的烧饼料理已经做了出来。这些烧饼是在一个长条形的容器里做出来的。
S切做完烧饼已经很累了，她想用最少的操作数使烧饼的画有笑脸的一面朝上。
这时，她手头有一个过长的烧饼翻转工具，这个烧饼翻转工具能正好翻动K个烧饼。也就是说，不改变烧饼先后顺序地直接翻转笑脸面→空白面；空白面→笑脸面；你不能翻转比K多或比K少的烧饼；长条形烧饼容器的两边有围栏，所以你也不能强行翻动比K少的烧饼。
作为远月学院的接班人创真，S切让你帮她想出这个最优操作，告诉她翻转所需的最少操作数或是这就是不可能完成的任务Mission Impossible。

输入格式

输入第一行是数据个数T，接下来跟着T个数据。
每个数据包括一行字符串S，和一个整数K，S代表长条形容器里烧饼的放置情况，每个字符不是 +（当前笑脸面朝上）就是-（当前空白面朝上）。

    数据范围
    1 ≤ T ≤ 100.
    Every character in S 不是 + 就是 -.
    2 ≤ K ≤ S的长度.

    其中50%的数据
    2 ≤ S的长度 ≤ 10.
    剩下50%的数据
    2 ≤ S的长度 ≤ 1000.


输出格式

对于每个测试数据，输出一行Case #x: y，x是第几个测试数据，y要么是 IMPOSSIBLE 如果根本没有办法用过长的烧饼翻转工具翻转所有烧饼变成全是笑脸面朝上的，要么输出一个整数代表最少的操作数。

样例数据

输入

3
---+-++- 3
+++++ 4
-+-+- 4

输出

Case #1: 3
Case #2: 0
Case #3: IMPOSSIBLE


备注

第一个样例，首先你翻转左边3个烧饼，变成 ++++-++-, 然后翻转最右边3个烧饼变成 ++++---+, 最后翻转剩下最后3个烧饼。（其他翻转方案都比3次多或次数相同）

第二个样例，没必要翻。

第三个样例，没法翻成全是+的。
